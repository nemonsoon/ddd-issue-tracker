# 学習ログ

Issue ごとに「予測」と「再説明」を残す場所。
合格条件①（ノートを見ずに再説明できる）の証拠になる。
なぜこの形式かは [learning-method.md](../learning-method.md) を参照。

## 使い方

1. Issue に着手する前に、[_template.md](_template.md) をコピーして `issue-<番号>-<slug>.md` を作る。
2. **着手前に「予測」ブロックだけ**埋める（ガイドを読む前。外れてよい）。
3. 実装して合格条件②（最小スライスが動く）を満たす。
4. ノートを閉じて「再説明」ブロックを見ずに埋める。
5. 「予測」と「再説明」の差分が、今回伸びた所。

## 重くしないルール

- 1 Issue 1 ファイル。長文にしない。
- きれいに書こうとしない。予測は外れていてよい（外れこそ学びの起点）。
- 実装の証拠はコミットハッシュ1行で足りる（コードを貼らない）。
